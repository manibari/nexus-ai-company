## 5. 交付檢查清單

PM Agent 在交付前必須確認：

### 文件完整性
- [ ] DELIVERY.md 已建立且內容完整
- [ ] QUICKSTART.md 已建立且內容完整
- [ ] 所有設計文件已更新為最終版本
- [ ] QA 測試計劃已標記所有測試通過

### 系統可用性
- [ ] 後端服務可正常啟動
- [ ] 前端服務可正常啟動
- [ ] 所有 API 端點可正常存取
- [ ] 核心功能可正常運作

### Demo 準備
- [ ] Demo 環境已準備就緒
- [ ] Demo 步驟已驗證可執行
- [ ] 測試資料已準備（如需要）

### 通知 CEO
- [ ] 建立 CEO To-Do（類型：review）
- [ ] 附上交付文件連結
- [ ] 說明 Demo 時間與方式

## 6. CEO To-Do 範本

當交付準備就緒時，PM 應建立以下 To-Do：

```json
{
  "type": "review",
  "title": "[專案名稱] 交付驗收",
  "description": "專案已完成 UAT，請驗收：\n\n1. 交付說明：[DELIVERY.md 路徑]\n2. 快速啟動：[QUICKSTART.md 路徑]\n3. Demo 網址：[URL]\n\n請依 DELIVERY.md 中的 Demo 指引進行驗收。",
  "priority": "high",
  "options": [
    { "id": "approve", "label": "✅ 驗收通過，結案" },
    { "id": "feedback", "label": "📝 需要修改" },
    { "id": "reject", "label": "❌ 退回重做" }
  ]
}
```
