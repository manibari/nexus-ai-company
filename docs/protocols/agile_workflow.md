> 版本：1.1 | 生效日期：2026-02-07 | 核准：CEO
>
> **整合系統：** [Product Board (ADR-012)](../decisions/ADR-012-product-board.md) + [CEO Inbox (ADR-015)](../decisions/ADR-015-ceo-todo-system.md)

---

## 1. 流程總覽

### 1.1 敏捷流程與 Product Board 對照

```
敏捷流程                    Product Board 階段
─────────────────────────────────────────────────
CEO 提需求     ──────────▶  P1 Backlog (需求池)
PM 拆任務      ──────────▶  P2 Spec Ready (規格確認)
SWE 開發       ──────────▶  P3 In Progress (開發中)
QA 測試        ──────────▶  P4 QA Testing (內部測試)
CEO 驗收       ──────────▶  P5 UAT (使用者驗收)
發布完成       ──────────▶  P6 Done (上線完成)
```

### 1.2 系統整合點

| 系統 | 用途 | 操作者 |
|------|------|--------|
| **CEO Inbox** | 需求提交、審批、驗收通知 | CEO |
| **Product Board** | 任務追蹤、階段管理、看板視圖 | PM, SWE, QA |
| **前端 URL** | http://localhost:3000 | 全員 |

## 2. 角色職責

| 角色 | 職責 |
|------|------|
| **CEO** | 提出需求、定義優先級、驗收 |
| **PM** | 需求分析、任務拆解、進度追蹤 |
| **SWE** | 技術設計、開發實作 |
| **QA** | 測試驗證、缺陷追蹤 |

## 3. Sprint 設定

### 3.1 Sprint 長度

**依專案彈性調整：**

| 類型 | 長度 | 適用 |
|------|------|------|
| 快速迭代 | 1 天 | 小功能、Bug 修復 |
| 標準開發 | 2-3 天 | 中型功能 |
| 大型功能 | 1 週 | 跨系統整合 |

### 3.2 單日 Sprint 時程

```
上午：PM 拆解任務 + SWE 開發
下午：SWE 完成 + QA 測試
結束：CEO 驗收（視需要）
```

---

## 4. 需求提交

### 4.1 對話方式

```
@PM 新需求

產品：[產品名稱]
需求：[需求描述]
優先級：P0/P1/P2/P3
```

### 4.2 User Story 方式

使用 `docs/templates/user_story_template.md`

### 4.3 優先級定義

| 優先級 | 意義 | 回應時間 |
|--------|------|----------|
| P0 | 緊急（阻擋發布） | 立即處理 |
| P1 | 高（必須完成） | 當日/當 Sprint |
| P2 | 中（盡量完成） | 下個 Sprint |
| P3 | 低（可延後） | Backlog |

---

## 5. 任務管理

### 5.1 狀態標記（對照 Product Board）

| 標記 | 狀態 | Product Board | 說明 |
|------|------|---------------|------|
| ⬜ | Backlog | P1_BACKLOG | CEO 提出的需求 |
| 📋 | To Do | P2_SPEC_READY | PM 完成規格 |
| 🔄 | In Progress | P3_IN_PROGRESS | SWE 開發中 |
| 🧪 | Testing | P4_QA_TESTING | QA 測試中 |
| 👀 | UAT | P5_UAT | CEO 驗收中 |
| ✅ | Done | P6_DONE | 完成發布 |
| ❌ | Blocked | BLOCKED | 受阻 |

### 5.2 任務估算

| 標記 | 工時 | 說明 |
|------|------|------|
| XS | < 2 小時 | 小修改 |
| S | 2-4 小時 | 簡單功能 |
| M | 4-8 小時 | 標準功能 |
| L | 1-2 天 | 複雜功能 |
| XL | > 2 天 | 需拆分 |

## 6. 進度更新

### 6.1 更新時機

**里程碑更新（預設）：**
- Sprint 開始時
- 任務完成時
- 遇到阻礙時
- Sprint 結束時

### 6.2 更新格式

```markdown
## [日期] 進度更新

### 完成項目
- [項目 1]

### 進行中
- [項目 2] (進度 XX%)

### 阻礙/風險
- [如有]

### 下一步
- [計劃]
```

---

## 7. 驗收流程

### 7.1 驗收方式（結合模式）

| 情境 | 驗收時機 |
|------|----------|
| P0/P1 重要功能 | 完成後立即驗收 |
| P2/P3 一般功能 | Sprint 結束統一驗收 |
| Bug 修復 | 修復後立即驗收 |

### 7.2 驗收標準

- [ ] 功能符合需求描述
- [ ] QA 測試通過
- [ ] 無 P0/P1 Bug
- [ ] 文件已更新（如需要）
